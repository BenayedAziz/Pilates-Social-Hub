    - useBrands() -> GET /api/brands

    - useChallenges() -> GET /api/challenges
    - useBingoCard() -> GET /api/bingo
    - useBadges() -> GET /api/badges
    - useLeaderboard() -> GET /api/leaderboard
    - useCalorieData() -> GET /api/calorie-data
    - useCircles() -> GET /api/circles
    - useWearable() -> GET /api/wearable
    - useStudioReviews(studioId) -> GET /api/studios/:id/reviews
    - useStudioCheckins(studioId) -> GET /api/studios/:id/checkins
    - useCurrentUser() -> GET /api/auth/me
  Mutations:
    - useCreatePost() -> POST /api/feed
    - useToggleLike() -> POST /api/feed/:id/like
    - useCreateForumPost() -> POST /api/forum
    - useVoteForumPost() -> POST /api/forum/:id/vote
    - useCreateBooking() -> POST /api/bookings
  Config: Bearer token auth from localStorage, staleTime 30s, retry 1

use-theme.ts:
  - Manages light/dark theme
  - Persists to localStorage ("pilates-theme")
  - Toggles "dark" class on <html> element
  - v2 migration: resets old dark preference to light

use-websocket.ts:
  - WebSocket client for real-time messaging
  - Auto-connects with JWT token from localStorage
  - Auto-reconnects every 5 seconds on disconnect
  - Event system: on(type, handler) for registering listeners
  - send(data) for outgoing JSON messages
  - connected state for UI indicator
  - Graceful error handling (silent fail if server doesn't support WS)

use-payments.ts:
  - useBookingPayment() -> POST /api/payments/booking (creates Stripe PaymentIntent)
  - useOrderPayment() -> POST /api/payments/order (creates Stripe PaymentIntent)
  - Returns { clientSecret, paymentIntentId }

--- CONTEXTS ---

AuthContext (AuthProvider + useAuth):
  State:
    - user: AuthUser | null (id, name, email, initials, level, bio)
    - isAuthenticated: boolean
  Actions:
    - login(email, password) -> POST /api/auth/login, stores JWT
    - signup(name, email, password) -> POST /api/auth/signup, stores JWT
    - logout() -> clears token + onboarded + preferences from localStorage
    - updateProfile({name?, bio?, level?}) -> PUT /api/auth/profile
  Behavior:
    - On mount: tries to restore session from JWT via GET /api/auth/me
    - Falls back to MOCK_USER (Emma D) if no token or API unavailable
    - Converts API response (displayName) to frontend shape (name + initials)
    - Shows nothing while initializing (prevents flash of login page)

AppContext (AppProvider + useApp):
  State:
    - Cart: cartItems[], addToCart, removeFromCart, clearCart, cartCount, cartTotal
    - Feed: likedPosts (Record), toggleLike, following (Record), toggleFollow
    - Community: votes (Record<id, "up"|"down"|null>), toggleVote
    - Store: wishlist (Set<number>), toggleWishlist
    - Booking: bookingSuccess (studioId | null), setBookingSuccess
    - Stats: currentStreak (12), longestStreak (18), totalSessions (47), totalCalories (12400)
    - Weekly: weeklyGoal (5), weeklyCompleted (4)
  Actions:
    - logSession(calories, duration) -> increments sessions, calories, weekly, streak

--- API ENDPOINTS (Express backend, 22 route modules) ---
================================================================================

HEALTH:
  GET /api/healthz -> { status: "ok" } (validated via api-zod schema)

AUTH:
  POST /api/auth/login -> { user, token } (bcrypt password check, JWT generation)
  POST /api/auth/signup -> { user, token } (creates user, hashes password)
  GET  /api/auth/me -> user object (requires Bearer token, authMiddleware)
  PUT  /api/auth/profile -> updated user (requires Bearer token)

STUDIOS:
  GET  /api/studios -> studio[] (supports ?q= and ?neighborhood= query params)
  GET  /api/studios/:id -> single studio
  GET  /api/studios/:id/reviews -> StudioReview[]
  GET  /api/studios/:id/checkins -> StudioCheckin[]

CLASSES:
  GET  /api/classes -> PilatesClass[] (supports ?studioId= and ?type=)

BOOKINGS (demo flow — real booking redirects to studio website):
  GET  /api/bookings -> Booking[] (user's demo bookings)
  POST /api/bookings -> new Booking (classId, studioId, timeSlot, paymentIntentId)

FEED:
  GET  /api/feed -> FeedPost[]
  POST /api/feed -> create new post (type, studio, duration, calories)
  POST /api/feed/:id/like -> toggle like

FORUM:
  GET  /api/forum -> ForumPost[] (supports ?category=)
  POST /api/forum -> create forum post (title, category, body)
  POST /api/forum/:id/vote -> upvote/downvote (direction: "up"|"down")

PRODUCTS:
  GET  /api/products -> Product[] (supports ?category=)

BRANDS:
  GET  /api/brands -> Brand[]

COACHES:
  GET  /api/coaches -> CoachProfile[]

CHALLENGES:
  GET  /api/challenges -> Challenge[]

BINGO:
  GET  /api/bingo -> BingoCell[]

BADGES:
  GET  /api/badges -> BadgeItem[]

LEADERBOARD:
  GET  /api/leaderboard -> LeaderboardEntry[]

CALORIE DATA:
  GET  /api/calorie-data -> CalorieDataPoint[] (8 weeks)

CIRCLES:
  GET  /api/circles -> PilatesCircle[]

MESSAGES:
  GET  /api/messages/conversations -> Conversation[]
  GET  /api/messages/conversations/:id -> { messages: Message[] }
  POST /api/messages/conversations/:id -> send message ({ content })
  GET  /api/messages/unread-count -> { count: number }

REVIEWS:
  GET  /api/studios/:id/reviews -> StudioReview[]
  POST /api/studios/:id/reviews -> create review (requires auth)

WEARABLES:
  GET  /api/wearable -> WearableData (recovery, strain, HRV, sleep, etc.)

ORDERS:
  POST /api/orders -> create order (items[], shippingInfo, totalAmount; optional auth)

PAYMENTS:
  GET  /api/payments/status -> { enabled: boolean, mode: "live"|"mock" }
  POST /api/payments/booking -> Stripe PaymentIntent for booking (requires auth)
  POST /api/payments/order -> Stripe PaymentIntent for order (requires auth)

UPLOADS:
  POST /api/uploads/image -> upload image to S3 (requires auth, multer, max 5MB images only)

WEBSOCKET:
  ws:// or wss:// on /ws?token=JWT
  Events:
    - "new_message" (incoming messages)
    - "typing" (typing indicators)
    - "message" (outgoing messages)

COMMON PATTERNS:
  - All routes use in-memory mock data as fallback when DATABASE_URL is not set
  - Database access via Drizzle ORM when PostgreSQL is available
  - JWT auth via authMiddleware (Bearer token)
  - optionalAuth middleware for endpoints that work with or without auth
